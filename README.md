# Классификация уровня IT-разработчика (junior / middle / senior) по резюме hh.ru

Этот проект — **proof of concept (PoC)**: небольшой эксперимент, который показывает, что по данным резюме hh.ru можно автоматически отличать уровни разработчиков **junior/middle/senior** с разумным качеством.

**Что делаем:**
1. Из `hh.csv` выбираем резюме IT-разработчиков.
2. Формируем целевую переменную `y = {junior, middle, senior}` эвристиками по названию должности и опыту.
3. Строим график баланса классов.
4. Обучаем классификатор по признакам резюме (зарплата, город, возраст, опыт, текст должности).
5. Печатаем `classification report` и делаем выводы о качестве.

## Запуск

### Установка зависимостей
```bash
python -m venv .venv
pip install -r requirements.txt
python app.py path/to/hh.csv
```

### Вывод

Модель показывает, что идея рабочая: по резюме hh.ru можно достаточно уверенно отличать junior/middle/senior, и качество на тесте получается разумным (общая точность около 0.85, лучше всего распознаётся senior). Ошибки в основном связаны с тем, что разметка уровня сделана эвристиками и неизбежно шумная, классы несбалансированы, а границы между middle и соседними уровнями размыты, поэтому именно middle чаще всего путается с junior и senior. Для улучшения качества больше всего помогут более аккуратная разметка части данных и расширение признаков за счёт нормализованных навыков и техстека.
